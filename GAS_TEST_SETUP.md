# 🚀 GAS版テストセットアップ手順

## 📋 セットアップ手順

### 1. Google Apps Script プロジェクト作成

1. **https://script.google.com** にアクセス
2. 「新しいプロジェクト」をクリック
3. プロジェクト名を「Threads投稿マネージャー」に変更

### 2. ファイルの設定

#### Code.gs
1. デフォルトの`Code.gs`の内容を全て削除
2. `/gas-version/Code.gs`の内容をコピー＆ペースト

#### dashboard.html
1. 「ファイル」→「新規」→「HTMLファイル」
2. ファイル名を「dashboard」に設定
3. `/gas-version/dashboard.html`の内容をコピー＆ペースト

### 3. 初期設定の実行

```javascript
// Apps Script エディタで以下の関数を実行

function initialSetup() {
  // 1. データ用スプレッドシートを作成
  const spreadsheetId = setupSpreadsheet();
  
  // 2. 自動実行トリガーを設定  
  setupTriggers();
  
  // 3. 結果をログに出力
  console.log('セットアップ完了');
  console.log('スプレッドシートID:', spreadsheetId);
}
```

### 4. 環境変数設定

「プロジェクトの設定」→「スクリプトプロパティ」で以下を追加：

```
THREADS_CLIENT_ID = your_threads_client_id
THREADS_CLIENT_SECRET = your_threads_client_secret
```

### 5. ウェブアプリとしてデプロイ

1. 「デプロイ」→「新しいデプロイ」
2. 「種類の選択」→「ウェブアプリ」
3. **実行ユーザー**: 自分
4. **アクセス権限**: 誰でも（匿名）
5. 「デプロイ」をクリック
6. **生成されたURLをコピー**

## 🧪 テスト手順

### 基本機能テスト

#### 1. ダッシュボード表示
- デプロイURLにアクセス
- ダッシュボードが正常に表示されることを確認

#### 2. アカウント追加テスト
1. 「アカウント管理」ボタンをクリック
2. 「新しいアカウントを追加」をクリック
3. テストデータを入力：
   ```
   アカウント名: テストアカウント
   ユーザー名: test_user
   アクセストークン: test_token_123
   ```
4. 「追加」をクリック
5. アカウントが追加されることを確認

#### 3. 投稿作成テスト（モック）
1. 「+」FABボタンをクリック
2. 投稿フォームに入力：
   ```
   投稿アカウント: テストアカウント
   投稿タイプ: テキスト
   投稿内容: これはテスト投稿です
   投稿タイミング: 今すぐ投稿
   ```
3. 「投稿」をクリック
4. ※実際のThreads API接続なしではエラーが発生（正常）

#### 4. スケジュール投稿テスト
1. 投稿フォームで「スケジュール投稿」を選択
2. 1時間後の時刻を設定
3. 投稿内容を入力して実行
4. スケジュール確認で登録されることを確認

### データ確認

#### スプレッドシートでの確認
1. Google Driveで「Threads投稿マネージャー - データ」を開く
2. 以下のシートが作成されていることを確認：
   - `accounts`
   - `scheduled_posts`  
   - `post_history`

#### ログ確認
1. Apps Script エディタで「実行時ログ」を確認
2. エラーやデバッグ情報を確認

## 🔧 トラブルシューティング

### よくある問題

#### 1. 権限エラー
**解決方法**: 初回実行時に権限を許可

#### 2. スプレッドシートアクセスエラー  
**解決方法**: `setupSpreadsheet()`を再実行

#### 3. トリガーが動作しない
**解決方法**: 
```javascript
function checkTriggers() {
  const triggers = ScriptApp.getProjectTriggers();
  console.log('登録されているトリガー:', triggers.length);
}
```

#### 4. データが表示されない
**解決方法**: ブラウザの開発者ツールでネットワークエラーを確認
